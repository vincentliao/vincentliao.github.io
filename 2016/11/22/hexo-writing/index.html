用 Hexo 寫東西

發表於 2016-11-22

最近同事推薦採用 Hexo 來當 Test Corner 的部落格系統，聽他說很符合需求也很多整合功能，而且開發者 Tommy Chen 是台灣人，當然要支持一下本地開發者。

簡單來說 Hexo 是個靜態的網頁產生器，可以吃 Markdown 後產生網頁存放在網頁伺服器端，加上現在很流行在 Github 上弄個 Github Pages 來當成個人或組織的網頁，所以將 Hexo 搭配著 Github Pages 透過 xxx.github.io 網址已經是很常見的組合。

其實已經有不少人寫過相關的使用方式，官方也很用心寫了完整的文件，基本上照做不太會有問題，我只是根據我所碰到的問題記錄下來。

開始寫文章

設定一切順利，那你可以開始寫文章，整個流程是這樣：

  • 先建立文件(hexo new)
  • 然後開始編寫
  • 產生靜態文件(hexo generate)
  • 啟動本地伺服器(hexo server --draft --open)，檢查看看樣子
  • 傳送到伺服器(hexo deploy)，讓所有人看到文章

後來想想，上面這個方法，倘若在別台已經編輯後，就會造成兩邊不同步的問題，在編輯前，怎麼取回最新的？總不能一直依賴複製吧？所以其實這方法並不好，或許可以在某一台機器上建立好環境，然後架設好 SSH Server，遠端登入那邊去編輯，其實就會是同一份，沒有資料同步問題。不然就是把整個檔案，除了那些生成的資源檔案以外，全部變成 git repo，然後放到 git server。需要另外的編寫環境時，就 git clone 回來，把 Hexo 安裝好，應該也可以，只是不知道哪個比較正規。
